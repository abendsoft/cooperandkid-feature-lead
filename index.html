Feature Lead Bridge

Customers still use https://esa.cooperandkid.com/apps/feature-lead on your store; Shopify forwards that to your hosted app (e.g. Vercel’s *.vercel.app — no DNS records on your domain).

For Shop domain, use cooperandkid.myshopify.com. Set FEATURE_LEAD_ADMIN_TOKEN, SHOPIFY_ADMIN_ACCESS_TOKEN, and optional SHOPIFY_SHOP_DOMAIN in your host’s environment (Vercel → Project → Settings → Environment Variables).

Partner app: get SHOPIFY_ADMIN_ACCESS_TOKEN (not shown in Dev Dashboard)
The dashboard only shows Client ID + Client secret. The Admin token comes from OAuth once: (1) In Vercel add OAUTH_STATE_SECRET (any long random string), SHOPIFY_API_KEY (Client ID), SHOPIFY_API_SECRET (Client secret), optional SCOPES (comma list, same as your app). (2) Redeploy. (3) Visit (replace shop if needed). (4) Approve; copy the token into SHOPIFY_ADMIN_ACCESS_TOKEN and redeploy. Use SHOPIFY_APP_URL if your public URL is not VERCEL_URL.

If rows stay empty after testing: open the live storefront (or theme preview), DevTools → Network, filter feature-lead, click Continue, and check the POST status (200 vs 401). In the theme editor, the preview runs in an iframe: the top-level console’s sessionStorage does not apply to the bridge script. Use a preview URL query like ?flb_debug=1&flb_preview_email=test@example.com on the storefront URL, or open DevTools from inside the preview frame. A 401 usually means SHOPIFY_APP_PROXY_SECRET does not match the app’s Client secret in the Partner Dashboard.
